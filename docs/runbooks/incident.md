# 障害対応フロー

VBA マクロ実行時に問題が発生した場合の対応手順。

## 緊急対応

### マクロが応答しない（ハングアップ）
1. `Esc` キーを押して中断を試みる
2. 中断できない場合は `Ctrl + Break`（Windows）/ `Cmd + .`（macOS）
3. VBA エディタが開く → 「終了」ボタンをクリック
4. `Application.ScreenUpdating = True` を「イミディエイト ウィンドウ」で実行

### 画面が更新されない
マクロ実行後に画面がフリーズしたように見える場合：
1. VBA エディタを開く（`Alt + F11`）
2. 「イミディエイト ウィンドウ」（`Ctrl + G`）で以下を実行：
```vba
Application.ScreenUpdating = True
Application.EnableEvents = True
Application.Calculation = xlCalculationAutomatic
```

### データが消えた・壊れた
1. **即座に Ctrl+Z（元に戻す）を試みる**
2. 元に戻せない場合は、ファイルを閉じる（保存しない）
3. バックアップや自動保存から復元する

## よくある問題と対処

### SheetManager

| 症状 | 原因 | 対処 |
|------|------|------|
| エラー 91: オブジェクト変数が設定されていない | ワークブックが開かれていない | 対象ブックを開いてから実行 |
| 「PERSONAL.XLSB には実行できません」 | PERSONAL がアクティブ | 対象ブックをクリックしてアクティブにする |
| 行が重複して挿入される | 2回目以降の実行で戻るリンクの判定失敗 | シート一覧を手動削除してから再実行 |

### FormatWorkbook

| 症状 | 原因 | 対処 |
|------|------|------|
| フォントが変わらない | Meiryo UI がインストールされていない | フォントをインストール、または `targetFont` を変更 |
| エラーダイアログが表示される | 保護されたシートが含まれている | シート保護を解除してから実行 |
| Zoom が変わらない | シートが `.Activate` されていない | 通常は自動で処理される。手動で Zoom 設定 |

## エスカレーション

上記で解決しない場合：
1. エラー番号とメッセージをメモする
2. 問題が発生したシートの状態（名前、データ量、グラフ有無）を記録する
3. GitHub Issues に報告する
