# アーキテクチャ概要

## システム構成

vba-tools は Excel VBA マクロの集合体で、機能ごとにフォルダを分けて管理する。各フォルダが1つの独立したツールとなる。

## コンポーネント

### sheet-manager（シート管理ツール）

**目的**: ブック内のシート一覧を自動生成し、ナビゲーションを提供する。

| ファイル | 種類 | 役割 |
|---------|------|------|
| SheetManager.bas | 標準モジュール | `UpdateSheetList` — 一覧シート生成 + 各シートへの戻るリンク追加 |
| ThisWorkbook.cls | クラスモジュール | `Workbook_NewSheet` — 新シート作成時に自動で一覧を更新 |

**データフロー**:
```
ユーザー実行 or 新シート作成イベント
  → UpdateSheetList 呼び出し
  → 既存の「シート一覧」シートを削除
  → 新しい「シート一覧」シートを先頭に作成
  → 各シート名をハイパーリンク付きでリスト化
  → 各シートの A1 に「シート一覧に戻る」リンクを追加（既存リンクは上書き、なければ行挿入）
```

**注意点**:
- 行挿入が発生するため、初回実行とリンク再作成で動作が異なる
- `PERSONAL.XLSB` への操作は明示的にブロック

### format-workbook（書式統一ツール）

**目的**: ブック全体のフォント・表示倍率を一括統一する。

| ファイル | 種類 | 役割 |
|---------|------|------|
| FormatWorkbook.bas | 標準モジュール | セル・テーブル・グラフ・図形のフォントを Meiryo UI に統一、Zoom を 80% に設定 |

**データフロー**:
```
ユーザー実行
  → 全シートをループ
    → セルのフォント統一
    → テーブル（ListObject）のフォント統一
    → グラフのフォント統一（タイトル・軸・凡例・データラベル）
    → 図形のフォント統一
    → Zoom 80% 設定
    → A1 セルを選択
  → 最初のシートをアクティブ化
  → 完了メッセージ表示
```

**注意点**:
- `ThisWorkbook` を参照（`ActiveWorkbook` ではない）
- グラフ・図形へのアクセスは `On Error Resume Next` で保護（存在しない場合の回避）
- Zoom 設定には事前の `.Activate` が必須

## 技術スタック

| 要素 | 技術 |
|------|------|
| 言語 | VBA (Visual Basic for Applications) |
| 実行環境 | Excel（macOS / Windows） |
| ファイル形式 | `.bas`（標準モジュール）、`.cls`（クラスモジュール）|
| 文字エンコーディング | UTF-8 |
| バージョン管理 | Git |

## 設計原則

1. **機能単位のフォルダ分離** — 1フォルダ = 1ツール。各フォルダに README.md を持つ
2. **定数化** — シート名などの固定文字列は `Private Const` で管理
3. **安全性** — `Application.ScreenUpdating` / `EnableEvents` / `Calculation` の保存・復元
4. **PERSONAL.XLSB 保護** — アドインブックへの誤操作を防止
5. **旧版保存** — 変更前のコードは `old/` フォルダに残す
