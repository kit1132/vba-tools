# VBA Release スキル

## リリース前チェックリスト

### 1. コード品質チェック

- [ ] 全 `.bas` / `.cls` ファイルに `Option Explicit` があるか
- [ ] デバッグ用コード（`Debug.Print`, テスト用 `MsgBox`）が残っていないか
- [ ] ハードコード文字列が定数化されているか
- [ ] `Application.ScreenUpdating` / `EnableEvents` / `Calculation` が正しく復元されるか

### 2. ドキュメントチェック

- [ ] 各フォルダの `README.md` が現在のコードと一致しているか
- [ ] ファイル構成表が実際のファイルと一致しているか
- [ ] ルートの `README.md` のフォルダ構成表が最新か
- [ ] マクロ一覧・機能説明が正確か

### 3. ファイル整合性チェック

- [ ] `.bas` / `.cls` ファイルが **UTF-8** で保存されているか
- [ ] 不要な一時ファイル・バックアップが含まれていないか
- [ ] `.gitignore` が適切に設定されているか
- [ ] `old/` フォルダに旧版が正しく保存されているか

### 4. 動作確認（ユーザーが Excel 上で実施）

- [ ] SheetManager: 新規ブックで `UpdateSheetList` を実行し、一覧シートが生成されるか
- [ ] SheetManager: 各シートに「戻る」リンクが追加されるか
- [ ] SheetManager: 2回実行して重複リンクが生じないか
- [ ] FormatWorkbook: テーブル・グラフ・図形を含むブックで実行し、フォントが統一されるか
- [ ] FormatWorkbook: Zoom が 80% に設定されるか
- [ ] FormatWorkbook: エラーハンドラが正しく動作するか（存在しないオブジェクトで実行）

## リリース手順

### 1. バージョン決定
- コミット履歴から変更内容を確認: `git log --oneline -10`
- 変更の性質に応じてコミットメッセージを整理

### 2. 最終コミット
```bash
git add -A
git status  # 不要なファイルが含まれていないか確認
git commit -m "リリース準備: [変更内容の要約]"
```

### 3. タグ付け
```bash
git tag -a v[X.Y.Z] -m "[リリース内容の要約]"
git push origin main --tags
```

### バージョニング規則
- **メジャー**: 新しいツール（フォルダ）の追加
- **マイナー**: 既存ツールへの機能追加
- **パッチ**: バグ修正、ドキュメント更新

## 制約

- リリース前に必ずチェックリストを全項目確認する
- `git push --force` は禁止
- main ブランチへの直接プッシュは確認を促す
