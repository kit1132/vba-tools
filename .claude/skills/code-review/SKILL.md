# VBA Code Review スキル

## レビュー手順

### 1. 事前確認
- 対象ファイルを Read ツールで全文読み込む
- 対象モジュールの README.md を確認し、仕様を把握する
- 既存の動作実績があるかユーザーに確認する

### 2. チェック項目

#### 必須チェック（全ファイル共通）
- [ ] `Option Explicit` が先頭にあるか
- [ ] 変数は `Dim` で型を明示しているか
- [ ] マジックナンバー・ハードコード文字列がないか（`Private Const` を使うべき）
- [ ] `Application.ScreenUpdating` を抑制→復元しているか
- [ ] `Application.EnableEvents` を変更している場合、必ず復元しているか
- [ ] `Application.Calculation` を変更している場合、必ず復元しているか

#### エラーハンドリング
- [ ] `On Error Resume Next` の範囲が最小限か
- [ ] `On Error Resume Next` の後に `On Error GoTo 0` で復元しているか
- [ ] エラーハンドラ（`On Error GoTo ErrHandler`）がある場合、`Cleanup` セクションで状態を復元しているか

#### シート操作（sheet-manager 固有）
- [ ] `PERSONAL.XLSB` への操作をブロックしているか
- [ ] シート削除前に `DisplayAlerts = False` → 削除後に `True` に戻しているか
- [ ] ハイパーリンクの `SubAddress` でシート名を `'` で囲んでいるか

#### 書式操作（format-workbook 固有）
- [ ] `ThisWorkbook` と `ActiveWorkbook` の使い分けが正しいか
- [ ] グラフ・図形へのアクセスで `On Error Resume Next` を使っているか（存在しない場合のエラー回避）
- [ ] Zoom 設定前に `.Activate` を呼んでいるか

### 3. レビュー結果の出力形式

各指摘を以下の形式で出力する：

```
### 指摘 #N: [タイトル]
- **リスク**: 高（動作に影響）/ 低（見た目・可読性のみ）
- **種類**: 構造変更 / ロジック変更 / 命名・コメント改善 / バグ修正
- **該当箇所**: ファイル名:行番号
- **問題**: 何が問題か
- **修正案**: どう直すべきか
```

### 4. 制約

- レビュー時にコードは変更しない（修正はユーザー承認後に別途実行）
- 動作確認済みのエラーハンドリングパターンへの変更提案は「リスク: 高」と明記する
- `old/` フォルダ内のコードはレビュー対象外（参考保存のため）
