# VBA Debug スキル

## デバッグ手順

### Step 1: 情報収集

ユーザーに以下を確認する：

1. **エラーメッセージ**: エラー番号と内容（VBA の `Err.Number` と `Err.Description`）
2. **再現手順**: どの操作でエラーが発生するか
3. **環境**: Windows / macOS、Excel のバージョン
4. **対象ブック**: シート数、テーブル・グラフ・図形の有無

### Step 2: 仮説立案

このプロジェクトでよくある問題パターン：

| パターン | 症状 | 原因 |
|---------|------|------|
| オブジェクト未設定 | エラー 91 | シート・グラフ・図形が存在しないのにアクセス |
| 型不一致 | エラー 13 | `Variant` を型指定変数に代入 |
| インデックス範囲外 | エラー 9 | 存在しないシート名・インデックスを参照 |
| フォントなし | エラー無視で適用されない | `Meiryo UI` が macOS にインストールされていない |
| 権限エラー | エラー 1004 | 保護されたシートへの書き込み |
| PERSONAL.XLSB | 意図しない変更 | `ActiveWorkbook` が PERSONAL を指している |

### Step 3: 原因特定

1. 対象ファイルを Read ツールで全文確認
2. エラー発生箇所を推定（エラー番号 + 行番号から特定）
3. `On Error Resume Next` で握りつぶされていないか確認
4. 変数のスコープ・ライフタイムを確認

### Step 4: 修正案の提示

修正案を以下の形式で提示する：

```
### 修正案
- **原因**: なぜエラーが発生したか
- **該当箇所**: ファイル名:行番号
- **修正前**: 現在のコード
- **修正後**: 提案するコード
- **リスク**: 高 / 低
- **テスト方法**: どうやって修正を確認するか
```

### Step 5: テスト手順の提示

修正後にユーザーが Excel 上で確認すべき手順を具体的に記載する：

1. VBA エディタでコードを差し替える手順
2. 実行前のバックアップ推奨
3. テストケース（正常系 + エラー発生時の確認）

## デバッグ用コード片

### 変数の値を確認
```vba
Debug.Print "変数名: " & 変数名
```

### 処理の到達確認
```vba
MsgBox "ここまで到達", vbInformation, "デバッグ"
```

### エラー情報の詳細表示
```vba
MsgBox "Error " & Err.Number & ": " & Err.Description & vbCrLf & _
       "Source: " & Err.Source, vbCritical
```

## 制約

- 修正はユーザー承認後に適用する
- 動作実績のあるエラーハンドリングパターンは変更しない
- デバッグ用コード（`Debug.Print`, `MsgBox`）はテスト後に削除を推奨する
